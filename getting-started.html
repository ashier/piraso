Overview

Piraso provides developer with real time application context information. Developers can do debugging, performance validation and monitoring to all environments even on production.

Context Monitoring

Monitor the traffic you want! Piraso lets you choose which traffic you want to monitor. You can monitor the traffic you yourself is making or others by simply providing their IP Address.

Piraso provides the following context information for a web piraso enabled application.

  • 1 Http request and response
  • 2 Application logging
  • 3 Database communication

Debugging

Supports debugging to the specific environment you really want to debug to. Developers usually need to recreate scenarios on their local machines before they will be able to see what is happening. With this you saves a lot of time. Right now debugging are supported for the following:

  • Log4j Debugging

    • All log levels and categories supported.
    • Specify context to your logging.
    • Supports for debugging of library dependencies. You can monitor hibernate, spring or other libraries you wish to watch.
    • Stack trace

  • SQL Debugging

    • Monitor all SQLs executed
    • Know all your classes that actually invokes the SQL being executed.
    • Multi datasource support
    • Stack trace

Performance Tuning

Provides elapse times which can be used to check performance.

  • Request-response turn around time (server-side)
  • Logs that supports method call displays elapse time
  • SQL Performance:
    • SQL explain plan
    • Execution time
    • Fetch time

Learning

Piraso can also serve as fast way to let you learn a particular application.

  • Developers monitoring SQLs will be available to learn more on entities used by a particular requests.
  • Discover and learn all supported services a particular application invokes.

Client Server

Piraso is composed of two components. The first component that resides on the web application and second, the client that lets you do the actual monitoring.

Server

This component allows you to enable your web application to support piraso monitoring. You as a developer can decide which feature will be available for monitoring.
